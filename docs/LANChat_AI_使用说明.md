# LANChat 聊天室 AI 助手 — 使用说明

> 场景组装 Agent v2 · 自然语言驱动的 3D 场景生成与协同设计

---

## 快速开始

### 1. 开房
在浏览器控制台（F12）中执行：
```javascript
LANChat.start_room({ room: "1024", password: "" })
```

### 2. 添加 AI 助手
```javascript
LANChat.add_agent({ name: "小黄", persona: "" })
```
`persona` 留空表示全能力「场景设计大师」。也可以指定专业角色：
```javascript
LANChat.add_agent({ name: "灯光师", persona: "专业的场景灯光设计师" })
LANChat.add_agent({ name: "整理师", persona: "场景整理助手" })
```

### 3. 让局域网其他人加入
前端会显示本机 IP 和端口。别人在浏览器控制台执行：
```javascript
LANChat.join_room({ ip: "192.168.x.x", port: 8770, room: "1024", password: "", nickname: "张三" })
```

### 4. 开始对话
在聊天输入框发送消息。多人 / 多 Agent 房间默认使用 `@助手名` 指定目标助手，例如 `@学者 帮我整理方案`。

规则：
- 只有一个本地 AI 助手时，可以尝试不写 `@`，系统可能会隐式触发该助手。
- 有多个 AI 助手时，必须显式 `@指定助手`；不 `@` 不会群发给所有助手。
- 不确定该找谁时，使用 `@GM`，由 GM 整理冲突、排序和确认项。

## 核心功能

### 一、场景组合（一键生成整个房间）

这是最核心的能力——说一句话，AI 自动完成「方案规划 → 3D 建模 → 智能布局 → 场景组装」全流程。

**触发方式**（多人 / 多 Agent 房间建议显式 `@助手名`）：
```
@小黄 生成欧式风格卧室
@小黄 生成一个现代客厅
@小黄 按清单生成场景
@小黄 组合整个卧室
```

**流程**：
1. AI 提取物体清单（双人床、床头柜、衣柜、梳妆台…）
2. 每个物体 → 文生图 → 图生 3D（Hunyuan3D）
3. 生成房间盒子（半透明六面体，5m×3m×3m）
4. LLM 智能布局（用物体真实 AABB 尺寸计算位置）
5. 批量导入引擎
6. 物理沉降 + 地面整平 → 所有物体整齐贴地

**回复示例**：
```
🏗️ 场景组合完成
  • 识别物体：12 个
  • ⚠️ 单次生成上限 8 个，本次先做前 8 个
  • 获取模型：8 个
  • 导入引擎：8 个
✅ 已放入场景：双人床、床头柜、衣柜、梳妆台、台灯、地毯、窗帘、全身镜
```

**注意事项**：
- 单次上限 8 个物体（防止生成时间过长），超出部分可以后续补充
- 天花板灯、窗帘、挂画、背景墙等会自动被过滤（不属于独立建模物体）
- 如果列表里有"床垫/被子/空调出风口/收纳篮"等琐碎项，会自动剔除

### 二、单步场景编辑

对已生成的场景做增删改。

**触发方式**：单 Agent 房间可直接说；多 Agent 房间请显式 `@助手名`：
```
@小黄 加个台灯                      → 在场景中新增台灯
@小黄 在沙发旁边放个落地灯           → 在特定位置放置
@小黄 把茶几往右移 0.5m             → 移动已有物体
@GM   删除旧椅子                    → 涉及删除/冲突时由 GM 提案并请求房主确认
@小黄 放大茶几                      → 修改物体尺寸
```

**回复示例**：
```
[场景设计大师] ✅ 已放置「台灯」到 (1.2, 0.0, -0.5)
[场景设计大师] 📋 已规划「落地灯」位置 (2.0, 0.0, 0.8)（模型已就绪）
```

### 三、普通聊天

AI 助手可以聊天、回答问题、提供设计建议。

```
@小黄 你好，你能做什么？
@小黄 欧式卧室一般用什么色调？
@小黄 帮我建议一下客厅布局
```

回复 `@小黄 /help` 查看完整能力列表。

---

### 四、讨论总结

多人讨论设计方案后，AI 可以总结共识并生成 Style Bible。

**触发方式**：
- 主动：`/总结` 或 `总结一下` 或 `来个总结`
- 自动：AI 判别讨论充分时主动询问"💡 需要我总结吗？"

**回复示例**：
```
📋 讨论总结
共识度: theme=85%  type=72%
Style Bible:
  主题: 欧式温馨
  色调: 米白、香槟金、奶油色
  材质: 实木、布艺、黄铜
  光照: 暖光 3000K
  避讳: 冷色调、工业风
场景名: 欧式主卧
确认后即可开始布置！
```

---

### 五、风格巡检

检查已生成的场景是否符合 Style Bible。

**触发**：`/检查` 或 `巡检一下`

**回复示例**：
```
🔍 风格巡检报告 (一致性: 85/100)
偏离项:
  ⚠️ 台灯_03: 冷白光源与暖色调不协调
    💡 建议改用 3000K 暖色光源
  ⚠️ 挂画_01: 工业风画面与欧式风格冲突
    💡 建议替换为古典油画
```

---

### 六、Plan 协商（讨论 → 定稿）

多人协作时，一人说「执行」即可进入 Plan 模式。

**流程**：
```
任何人: 执行
  → AI: 📋 已根据讨论整理出方案（v1）
         物品清单：双人床、床头柜、衣柜…
         这样可以吗？回复「确认」定稿或提出修改意见。

用户: 衣柜换成通顶的
  → AI: 📝 方案已更新（v2）

用户: 确认
  → AI: ✅ 方案已定稿。如需生成，请说「开始执行/生成场景」。
```

**关键行为**：
- 新意见 → 静默累积，积攒够再输出合并方案
- 否定 → 立即修订输出
- 确认 → 定稿（不自动执行，需手动说"开始执行"）

---

### 七、3D 模型自动导入

当 AI 回复中出现混元模型目录路径时，系统会自动等待 mesh 下载完成后导入引擎。

**无需手动操作**。回复中看到"模型目录：`models/hunyuan_xxx`"后，稍等片刻即有系统消息：
```
🎉 已将生成的 3D 模型导入场景：hunyuan_20260609_000600
```

## 高级功能

### 隐式触发（仅限单 Agent 房间）

以下类型消息在只有一个本地 AI 助手时可以自动触发 AI，不需要 @ 前缀：
- `加个/添加/删除/移动` 等场景编辑指令
- `生成3D模型/生成XX风格卧室` 等生成指令
- `总结一下/巡检一下` 等内置命令
- `按清单/布置场景/组合房间` 等组合指令

多 Agent 房间不做群体隐式触发。请显式 `@助手名`，或 `@GM` 让 GM 整理讨论和确认项。

普通闲聊（"你好""谢谢""好的"）**不会**自动触发，保持静默。

### 意图判别

每条用户消息都会经过意图分类：
- `execute` → 走场景编辑/组合流程
- `summarize` → 主动询问是否需要总结（带防抖，60s 内不重复）
- `none` → 普通聊天，不打扰

### 记忆增强

连续对同一类物体操作时，AI 会主动提示：
```
用户: 加个椅子  → ✅ 已放置
用户: 再加一个椅子 → ✅ 已放置
用户: 再来一个椅子 → ✅ 已放置
                    💭 你已连续添加了 3 个「椅子」类物体，需要按相同方式继续吗？
```

## 常见问题

### Q: 为什么我只看到一个物体进入场景，其他都没生成？
A: 如果消息是"生成 3d 模型"但没带 `@`，且不在场景组合触发词范围内，会走 CAIApp 内置 agent（一次只生成一个）。要批量生成整个房间，请使用"生成欧式风格卧室""按清单生成场景"等完整指令。

### Q: 物体在场景中散乱、不正？
A: 重启后应该修复了——现在有受控物理沉降 + 地面整平 + 房间盒子约束。如果仍散乱，查看日志中是否有 `[SceneComposer] 受控沉降+钳制+整平完成`。

### Q: 回复中出现了奇怪的错误文本？
A: 如果看到 `'str' object has no attribute 'model_dump'`，这是 DMXAPI 的间歇性兼容问题，**重发一次即可**。系统已对接友好降级。

### Q: 为什么单次只生成 8 个物体？
A: 单次上限 8 个，因为每个物体都要走「文生图 → 图生 3D」链路，耗时较长。超出部分可以说"继续生成剩下的"来补充。

### Q: 可以切换 AI 风格吗？
A: 可以。添加 Agent 时指定 `persona` 中的关键词即可自动匹配：
- 含"赛博朋克/cyberpunk" → 赛博朋克设计师
- 含"极简/北欧/minimalist" → 北欧极简设计师
- 含"中式/新中式/茶室" → 新中式设计师
- 含"工业风/industrial" → 工业风设计师
- 含"灯光/光照" → 灯光师
- 含"绿植/植物" → 绿植顾问

### Q: 局域网其他人连不上？
A: 检查两点：
1. 确保显示的本机 IP 是**真实局域网 IP**（如 192.168.x.x），不是虚拟机网卡地址
2. 确保 Windows 防火墙允许 8770 端口入站

## 技术参考

- **端口**：8770（WebSocket）
- **房间盒子尺寸**：默认 5m×3m×3m（宽×深×高）
- **单次物体上限**：8 个
- **总结防抖**：60 秒内不重复建议
- **物理沉降**：高阻尼 0.98 + 零弹性 0.0 + 0.6s 沉降
- **位置钳制**：离墙 0.15m 空隙
