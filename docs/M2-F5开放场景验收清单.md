# M2-F5 开放场景验收清单

> 目的：验证 M2 去特殊化不是离线 JSON 绿，而是在 F5 实机场景中成立。离线测试只能作为基础门槛，合并判断以本清单为准。

## 共同检查

- 先保存/截图 decompose JSON，确认生成差异来自 `ZoneAspect.params`，不是代码侧场景关键词推断。
- `compose()` 返回结果会包含 `zone_decompose_snapshot`；优先打开该 JSON，里面有 `raw_zones` 和 `normalized_zones`。
- 可用 `python docs/probes/m2_f5_snapshot_check.py <zone_decompose_snapshot> <scene_kind>` 做快速字段检查；`scene_kind` 可取 `grass_yurt / church / observatory / outdoor_market / auto`。
- 确认 `PHASE_ORDER` 行为仍为 `GROUND -> SHELL -> INTERIOR -> BOUNDARY -> OBJECTS -> DECORATION`。
- 确认无 `boundary` / `ground_cover` aspect 时不会生成围栏或草覆盖。
- 确认 `unsupported` 只进入报告/backlog，不进入执行派发。

## 场景 1：草原蒙古包

建议提示词：

```text
生成一个草原上的蒙古包营地，有蒙古包、低矮木围栏、草地和少量野花，内部有地毯和桌椅。
```

必须检查的 decompose JSON：

- 外层 terrain 有 `ground_profile.type=rolling`，`material=grass`，合理 `amplitude`。
- 有 `ground_cover`，且 `kind/scatter` 与草地一致。
- 有 `boundary`，`kind=fence` 或等价低矮木栏。
- shell 内部 `interior_surface.floor_shape=disc` 或缺省后仍保持 disc。

F5 视觉判据：

- 蒙古包贴地，不埋入、不悬空。
- 围栏仍套住蒙古包中心，不能被 terrain/zone center 平移带偏。
- 地面起伏、草/花、围栏比例与草原营地一致。
- 内部地面为圆/椭圆并贴边，不出现方角穿出圆壳。

## 场景 2：欧式教堂

建议提示词：

```text
生成一个欧式石质小教堂场景，教堂内部有石板地面和长椅，外部不要草地和围栏。
```

必须检查的 decompose JSON：

- 不应有 `ground_cover`。
- 不应有 `boundary`。
- shell 或室内 zone 有 `interior_surface.floor_material=stone`。
- 必须有 `interior_surface.floor_shape=quad`。
- 不应出现 `curtain`、`felt`、`毡帘` 等蒙古包入口提示。

F5 视觉判据：

- 无草、无栅栏、无毡帘。
- 室内地面是方形/矩形石板感，不应强行套圆盘。
- 地面贴合建筑 footprint，不明显露出或穿墙。

## 场景 3：火山口观测站

建议提示词：

```text
生成一个火山口边缘的科研观测站，有金属和混凝土质感，周围是深色岩地；如果需要岩浆流但当前能力不支持，请记录为 unsupported。
```

必须检查的 decompose JSON：

- `style_context.main_building` 指向观测站或科研建筑。
- `ground_profile.material=stone` 或 neutral/stone 类合理材料。
- 如出现 `lava_flow` / `steam` / `fog` 等未支持能力，必须进入 `unsupported`。
- 不应 fallback 成 grass/flowers。

F5 视觉判据：

- 场景不崩。
- 不出现默认草地。
- unsupported 不阻塞主链路。

## 场景 4：纯室外营地 / 集市 / 广场

建议提示词：

```text
生成一个没有可进入建筑的纯室外游牧集市，有几顶不可进入的小摊帐、篝火、货摊和一圈低矮木围栏。
```

必须检查的 decompose JSON：

- 只有 terrain 或 terrain + 非可进入小物件，不应生成可进入 shell 作为主建筑。
- 必须有 `boundary` aspect。
- `boundary.params.radius` 可显式给出；未给时检查 `zone.size` 与 `margin` 是否能得到合理边界范围。
- `boundary.kind/material/height` 与集市/营地风格一致。

F5 视觉判据：

- 无 shell 时仍能生成 boundary，这是 M2-2 的关键验收。
- 围栏相对物体分布合理：不切穿篝火、帐篷、货摊，也不能只是远离主体的一圈空边界。
- 如果围栏位置失败，优先调整 decompose 的 `zone.size` / `boundary.radius` / `margin`，不要回退到代码侧场景关键词。

## 通过标准

- 四个场景都能生成，不崩溃。
- 草原零回归成立：蒙古包、草、花、围栏、圆/椭圆内地面、贴地关系可接受。
- 教堂走同一套 capability 机制，但呈现出无草、无围栏、非毡帘、方形石板地。
- 陌生场景不会被猜成草地；unsupported 被记录。
- 纯室外场景证明 boundary 不再依赖 `_shell_aabb`。
