### M2 步骤 15b · interior_skin 地面（修 F5 蒙古包内物体悬空、地面全黑）

- **日期**：2026-06-14
- **文件**：[scene_composer.py](../editor/plugins/AITool/cai_extensions/agent/scene_composer.py)（`_build_floor_obj` 材质参数化；新增 `_generate_interior_floor`；`_generate_scene_framework` 给 shell zone 铺内皮地面）
- **改动**：
  - `_build_floor_obj(mtl_lib, mtl_name)` 材质参数化——terrain 用草地、shell 内皮用地毯，共用同一份已验证的 quad 几何（默认值保持 terrain 调用不变）。
  - 新增 `_generate_interior_floor(zone)`：为 shell zone 程序生成一块地面（地毯暖色 `Kd 0.55 0.28 0.20`），**内嵌 0.85×**——方地面边长取 volume 的 0.85，四角不戳穿圆壳墙体。略抬 1cm 防与 terrain z-fighting。材质从 `zone.interior_skin.floor_material` 读（参数化），缺省地毯。
  - `_generate_scene_framework`：shell zone 在 `_place_shells` 放外壳之外，额外调 `_generate_interior_floor` 补地面。
- **原因**：F5"草原蒙古包"现象——进蒙古包后地面全黑、物体（绿植）悬空（截图2）。根因：shell 建筑模型是实心外观团块，**没有可用内表面**（文档 §任务C G老师明确："interior_skin 必须参数化生成，不靠外壳内表面"）。
- **为何只补 floor 不补 wall/ceiling**：外壳模型从内侧已挡住大部分墙/顶视线，最朴素的 interior_skin 取值就是"地面+默认中性墙顶"。floor 是最影响观感的（物体有处可立）。wall/ceiling 留作后续扩展，数据结构（`InteriorSkin.wall_material/ceiling_material`）已预留。
- **避开已报备风险**：15a 提的"方内皮盒戳穿圆壳"——这里只铺地面（不是盒），且内嵌 0.85×，四角在圆壳半径内，不戳穿。
- **离线验证**：carpet 几何与 grass 字节一致（仅材质行不同）、ASCII 通过、Kd 暖色主导、内嵌数学（4m→3.4m）、shell 场景分发（has_shell→铺地面非白盒）。
- **验证状态**：已语法检查 + 离线验证通过，待 F5 目测蒙古包内有地毯、物体落地

### M2 步骤 15a · shell 从家具路径拆出当围合体（修 F5 双围合撕裂）

- **日期**：2026-06-14
- **文件**：[scene_composer.py](../editor/plugins/AITool/cai_extensions/agent/scene_composer.py)
  - decompose prompt 加 `enclosure=shell` + `shell_asset` 字段（标志性外观建筑 vs 中性 box 房间）
  - `_build_zone_tree`：shell zone 记 `primary_shell_asset_id`，**不开矩形门洞**（用模型自带入口）
  - 新增 `_collect_shell_assets()`（收集 shell asset 名）+ `_place_shells()`（确定性包裹放置）
  - `compose()`：注入并保护 shell 名（截断时保 shell）+ Phase 3 把 shell 从 resolved 分离
  - `_generate_scene_framework`：shell zone **不生成白盒**
  - `_run_original_workflow`：框架生成后调 `_place_shells`
- **修的是什么（F5 实锤）**："草原上的蒙古包" F5 暴露：蒙古包被 `extract_items` 当家具、布局 LLM 摆原点，同时 `_generate_room_box` 又在原点生成方盒+矩形门洞 → **两个围合体（圆壳 vs 方盒）不同心、双门不对齐 → 门区几何撕裂**（截图2 的破碎带状）。根因：没有 shell 路径，建筑只能走家具路径，必然冲突。
- **怎么修**：把"标志性外观建筑"识别成 `enclosure=shell`，走**确定性围合路径**——
  - 从家具清单剔除（不进布局 LLM，不被摆中心）
  - `_place_shells` 居中落地、按 AABB 缩放到包住 volume（WRAP=1.25 略大留白）
  - **用模型自带入口**（蒙古包门帘），不在圆模型上 punch 矩形洞 → 撕裂消失
  - shell zone **不生成白盒**（模型本身就是围合）
- **开放性保持**：shell vs box 的判断在 LLM decompose 那端（"标志性建筑"→shell，"普通房间"→box），代码不枚举（不写 if 蒙古包）。
- **零回归**：纯室内房间 decompose 仍输出单 box、无 shell → `_collect_shell_assets()` 空集 → 走旧白盒路径，字节一致。shell 只在"室外套标志性建筑"才触发。
- **离线验证**（ALL PASS）：树构建（shell 记 asset、不开门洞、box@原点）、`_collect_shell_assets`、shell 注入+截断保护、Phase3 分离（yurt 不进家具）、WRAP 缩放数学。
- **待 F5**（引擎侧，离线验不了）：蒙古包模型是否包住 volume、入口朝向、撕裂是否消失、furniture 是否落在壳内、纯室内零回归。
- **已知风险先报备**：① 圆壳 vs 方内皮盒四角可能戳穿（15b 内皮盒做内切）；② 入口朝向 rot=0 无法保证对齐 +Z，待 F5 看是否需转向；③ shell pivot 假设在底部中心，若模型 pivot 在几何中心会半埋地下。
- **验证状态**：已语法检查 + 五组离线验证全过；引擎侧待 F5
