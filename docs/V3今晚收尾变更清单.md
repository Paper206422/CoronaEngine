# V3 今晚收尾变更清单

日期：2026-06-19

## 1. 本轮目标

增强 V3 的正式测试稳定性和效率，优先服务今晚 F5 验收：

- 单人 / 多 Agent 交互不串路。
- 多人 / 多 Agent 协作不重复同步资源。
- 渐进式生成有持续、用户可理解的信息披露。
- 室内、室外、混合场景组装结果有导入/完成证据。
- 生成中介入必须有吸收、延后或失败说明。
- 日志分析可以快速落盘，减少人工翻日志时间。

## 2. 已落地改动

### 2.1 资源长耗时披露

- 图片补偿阶段增加安全心跳，长时间等待时提示“参考图片仍在准备中”。
- 3D 模型批次生成增加等待心跳，长时间等待时提示“第 N/M 批模型仍在生成”。
- 检索、降级、模型批次、入库等资源阶段通过安全 progress sink 披露给聊天室。
- 披露不包含 `job_id`、`session_id`、provider URL、API key、线程名等内部信息。

### 2.2 检索降级与资源失败语义

- Dashscope / embedding key 缺失时预检跳过图搜路径，避免反复 401。
- 检索 401 会快速熔断剩余检索任务并转生成。
- 图片补偿失败会转文字直生 3D。
- 缓存模型 / 缓存图片会跳过慢补偿。

### 2.3 批次资源补全

- 生成中新增对象可形成后续 `BatchResourcePlan`。
- 后续资源请求会先跑图片阶段，再跑模型检索/生成，再进入当前批次导入。
- 资源队列有上限和 backlog 披露，避免静默丢失。
- 缺模型 provider 时明确报失败，不要求用户提供模型路径。

### 2.4 多人同步稳定性

- host 周期快照不携带 actor create。
- 显式快照请求不会清空 `snapshotActorCreateKeys`，避免重复请求快照时重发所有 actor create。
- AI 场景框架 actor 白名单保留：`__room_terrain`、`__terrain_boundary` 等可同步，普通内部 actor 仍过滤。
- `docs/f5_reports/` 下正式运行报告被局部 `.gitignore` 忽略，避免污染 `git status`。

### 2.5 前端验收可读性

- LANChat 主体、气泡、输入框、披露条提升到约 15px 文本。
- 聊天气泡增加行高和 padding。
- 成员栏加宽，成员/AI 助手列表使用更清晰字号。
- Network 协作面板使用更可读的 `text-sm` 和较高输入框。
- 添加“夜市验证组”模板，便于一键添加长者、商人、小女孩、山贼。

### 2.6 F5 日志探针与报告

新增：

- `docs/probes/v3_f5_log_check.py`
- `docs/probes/v3_f5_quick_gate.py`
- `docs/probes/test_v3_f5_log_check.py`
- `docs/probes/test_v3_f5_quick_gate.py`

探针检查项：

- `actor-create`：重复 actor create 广播。
- `file-request`：重复资源请求，特别是 `Resource/terrain.obj`。
- `progress-gap`：用户可见进度长时间静默。
- `resource-heartbeat`：资源阶段心跳是否存在。
- `user-visible-leak`：用户可见消息是否泄露内部字段。
- `terrain-meshopt-noise`：解释 terrain mesh split 噪声。
- `cef-crash`：CEF / Chromium fatal / DCHECK 崩溃。
- `completion-integrity`：完成摘要和导入证据是否一致。
- `intervention-visibility`：生成中介入是否有吸收、延后或失败说明。
- 报告表格包含每个检查项的处置建议，现场看到 WARN/FAIL 后可直接按建议定位。

## 3. 今晚推荐命令

改完代码或首次验收前：

```powershell
python editor\plugins\AITool\services\verify_ultimate_plan.py
```

每轮 F5 后：

```powershell
python docs\probes\v3_f5_quick_gate.py --skip-verify --write-report
```

若要指定日志：

```powershell
python docs\probes\v3_f5_quick_gate.py --skip-verify --write-report --log build\examples\engine\RelWithDebInfo\logs\<run>_corona.log
```

## 4. 判读标准

- `F5_READY`：日志层无 WARN/FAIL，可作为该轮非 native 日志通过信号。
- `F5_REVIEW_WARNINGS`：没有阻断项，但需要人工确认 WARN 是否可解释。
- `F5_BLOCKED`：存在 FAIL，不建议签收该轮。

旧日志 `2026-06-19_02-52-51_corona.log` 的校准结果为：

- `F5_REVIEW_WARNINGS`
- terrain 文件只请求 1 次。
- MeshOpt terrain 日志是网格分段噪声，不是重复导入。
- completion-integrity 通过。
- 进度披露和资源心跳属于修复前缺口。

## 5. 仍需 F5 验证

- 新日志前 10 分钟是否持续出现资源状态或心跳。
- 新 peer 加入时是否不再重复重发所有 actor create。
- 生成中新增/调整是否在最终报告中给出处置结果。
- 室内、室外、混合场景的真实摆放、地形边界和视觉效果。
- CEF/Chromium 手势 DCHECK 是否仍会导致 UI 面板崩溃。
- VLM 开关打开时是否能稳定完成审查或明确降级。

## 6. 注意事项

- 今晚不要跑 CMake / Ninja / native build。
- `verify_ultimate_plan.py` 末尾的 401、VLM timeout、broken import 等堆栈是故障注入测试，只看最终 `[SUMMARY]`。
- 每轮 F5 报告写入 `docs/f5_reports/`，报告文件默认忽略，不进入提交。
